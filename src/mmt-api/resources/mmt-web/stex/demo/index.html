FoMTeX Demo

Imagine a user authoring a LaTeX document like:

If the square of some natural number $N$ is even, then $N$ is even.

Using semantic annotations, this fragment can be marked up (whether directly by the user or via NLP techniques) to provide additional semantic information, e.g. using this prototypical syntax:

\theorem{
  \vardecl[type={\NaturalNumber},forall]{\varN}{N}
  \implication{
    If \arg[1]{
      \even{
        \arg[1]{
          \natpow{\arg*[2]{2} the square of some \NaturalNumber{natural number}
		\arg[1]{$\varN$}
	  } is even
	}
      }
    }, then
      \arg[2]{
        \even{\arg[1]{$\varN$} is even}
      }
  }
}

The annotations are entirely invisible. Similarly, the .pdf generated from the LaTeX code should not look any differently than expected.

However, The MMT system can parse this file, extract the semantic annotations in it, and serve it as an active document.
